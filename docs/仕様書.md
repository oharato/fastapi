# 仕様書

## 1. API 仕様

### 共通

- **ベース URL**: `http://localhost:8000`
- **Content-Type**: `application/json`
- **文字コード**: UTF-8
- **認証**: `Authorization: Bearer <JWT>` ヘッダー（`/api/auth/login` を除く全エンドポイント必須）

---

### 1.0 認証 API

#### POST `/api/auth/login`

メールアドレスとパスワードでログインし、JWTトークンを取得する。

**リクエストボディ**

```json
{ "email": "user@example.com", "password": "password123" }
```

**レスポンス** `200 OK`

```json
{ "access_token": "eyJ...", "token_type": "bearer" }
```

**エラー** `401 Unauthorized` — 認証情報が不正

---

#### GET `/api/auth/me`

現在のログインユーザー情報を返す。

**レスポンス** `200 OK`

```json
{ "id": 1, "email": "user@example.com", "is_active": true }
```

**エラー** `401 Unauthorized`

---

### 1.1 TODO API

> **すべてのエンドポイントで認証必須。ユーザーは自分のTODOのみ操作できる。**

#### GET `/api/todos`

TODO の一覧を返す。

**レスポンス** `200 OK`

```json
[
  {
    "id": 1,
    "title": "牛乳を買う",
    "completed": false,
    "created_at": "2026-03-11T14:00:00Z"
  }
]
```

---

#### POST `/api/todos`

TODO を新規作成する。

**リクエストボディ**

```json
{ "title": "牛乳を買う" }
```

**バリデーション**: `title` は 1 文字以上 200 文字以下

**レスポンス** `201 Created` — 作成した TODO オブジェクト

---

#### PATCH `/api/todos/{todo_id}`

TODO を部分更新する。

**リクエストボディ**（すべてオプション）

```json
{ "title": "新しいタイトル", "completed": true }
```

**レスポンス** `200 OK` — 更新後の TODO オブジェクト
**エラー** `404 Not Found` — 指定 ID が存在しない場合

---

#### DELETE `/api/todos/{todo_id}`

TODO を削除する。

**レスポンス** `204 No Content`
**エラー** `404 Not Found`

---

### 1.2 ジョブ API

> **すべてのエンドポイントで認証必須。ユーザーは自分のジョブのみ操作できる。**

#### GET `/api/jobs`

ジョブ一覧を作成日時の降順で返す。

**レスポンス** `200 OK`

```json
[
  {
    "id": "42d1ff14-5ecf-4ea0-9573-86d55e20aeb8",
    "name": "sample",
    "params": "{\"seconds\": 10}",
    "status": "success",
    "progress": 100,
    "result": "10秒のジョブが完了しました",
    "error": null,
    "created_at": "2026-03-11T14:00:00Z",
    "started_at": "2026-03-11T14:00:00.1Z",
    "finished_at": "2026-03-11T14:00:10.2Z"
  }
]
```

---

#### GET `/api/jobs/stream`

ジョブ一覧を **Server-Sent Events（SSE）** でリアルタイム配信する。1秒ごとに全ジョブリストを送信する。

**認証**: `?token=<JWT>` クエリパラメータまたは `Authorization: Bearer` ヘッダー（EventSource はカスタムヘッダー不可なためクエリパラメータを推奨）

**レスポンス** `text/event-stream`

```
data: [{"id": "...", "status": "running", ...}]

data: [{"id": "...", "status": "success", ...}]
```

---

#### GET `/api/jobs/{job_id}`

単一ジョブの詳細を返す。

**レスポンス** `200 OK` — ジョブオブジェクト
**エラー** `404 Not Found`

---

#### POST `/api/jobs`

ジョブを新規投入する。

**リクエストボディ**

```json
{ "name": "sample", "params": { "seconds": 10 } }
```

| フィールド | 型 | 説明 |
|---|---|---|
| `name` | string | ジョブ種別名。`TASK_REGISTRY` に登録済みのもののみ有効 |
| `params` | object | タスク関数に渡すパラメータ |

**現在利用可能なジョブ種別**

| name | パラメータ | 説明 |
|---|---|---|
| `sample` | `seconds: int`（デフォルト: 10） | 指定秒数かけて進捗を更新するデモジョブ |

**レスポンス** `201 Created` — 作成したジョブオブジェクト（status は `pending`）
**エラー** `400 Bad Request` — 未登録のジョブ種別

---

#### DELETE `/api/jobs/{job_id}`

ジョブをキャンセルする。`pending` または `running` 状態のジョブのみキャンセル可能。

**レスポンス** `204 No Content`
**エラー** `404 Not Found` — 存在しない場合
**エラー** `400 Bad Request` — キャンセル不可能な状態（success/failed/cancelled）

## 2. データモデル

### User

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | int (PK, auto) | 主キー |
| `email` | string (unique) | メールアドレス |
| `hashed_password` | string | bcryptハッシュ化済みパスワード |
| `is_active` | bool | 有効フラグ（デフォルト: true） |
| `created_at` | datetime (UTC) | 作成日時（自動設定） |

### Todo

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | int (PK, auto) | 主キー |
| `user_id` | int (FK→User) | 所有ユーザー |
| `title` | string | タイトル（1〜200文字） |
| `completed` | bool | 完了フラグ（デフォルト: false） |
| `created_at` | datetime (UTC) | 作成日時（自動設定） |

### Job

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string (PK) | UUID（Taskiq タスク ID） |
| `user_id` | int (FK→User) | 所有ユーザー |
| `name` | string | ジョブ種別名 |
| `params` | string | JSON 文字列 |
| `status` | enum | `pending` / `running` / `success` / `failed` / `cancelled` |
| `progress` | int | 進捗 0〜100（デフォルト: 0） |
| `result` | string? | 成功時のメッセージ |
| `error` | string? | 失敗時のエラー詳細 |
| `created_at` | datetime (UTC) | 投入日時 |
| `started_at` | datetime? | 実行開始日時 |
| `finished_at` | datetime? | 完了/失敗/キャンセル日時 |

## 3. ジョブライフサイクル

```
[投入]
  POST /api/jobs
      ↓
  DB: status=pending
  Redis: タスクキューに追加 (broker.kiq)
      ↓
[ワーカー受信]
  DB: status=running, started_at=now
      ↓
[進捗更新（1秒ごと）]
  DB: progress=0→100
      ↓（キャンセルチェック）
  status=cancelled → 中断
      ↓（正常完了）
  DB: status=success, progress=100, result=..., finished_at=now
      ↓（例外発生）
  DB: status=failed, error=..., finished_at=now
```

## 4. 新しいジョブ種別の追加方法

1. `app/tasks/` に新しいタスクファイルを作成する

```python
# app/tasks/my_task.py
from app.broker import broker
from app.models.job import JobStatus
from app.models.user import User  # noqa: F401 - FK解決に必要
from app.services import job as job_service
from sqlmodel import Session
from app.database import engine

@broker.task
async def run_my_task(job_id: str, param1: str) -> str:
    with Session(engine) as session:
        job_service.update_job_status(session, job_id, JobStatus.running)
    # 処理...
    with Session(engine) as session:
        job_service.update_job_status(session, job_id, JobStatus.success, result="完了")
    return "done"
```

2. `app/routers/job.py` の `TASK_REGISTRY` に登録する

```python
from app.tasks.my_task import run_my_task

TASK_REGISTRY = {
    "sample": run_sample_job,
    "my_task": run_my_task,   # 追加
}
```

3. ワーカー起動コマンドに新モジュールを追加する

```bash
uv run taskiq worker app.broker:broker app.tasks.sample app.tasks.my_task
```

## 5. フロントエンド

### 共通（`/static/app.js`）

- `authFetch(url, options)` — Authorization ヘッダー付き fetch。401時にログインページへリダイレクト
- `getToken()` — localStorage のトークンを返す。未ログインならログインページへ
- `initTheme()` / `toggleTheme()` — DaisyUI テーマのlocalStorage永続化

### TODOリスト（`/`）

- Alpine.js でリアクティブな CRUD 操作を実現
- API: `/api/todos` に対して `authFetch`（GET / POST / PATCH / DELETE）
- フィルター状態はクライアント側で管理（`get filtered()`）

### ジョブ管理画面（`/jobs.html`）

- `EventSource('/api/jobs/stream?token=<JWT>')` でSSE接続し、1秒ごとにジョブ一覧を自動更新
- `destroy()` フックで EventSource を明示的にクローズし接続リークを防止
- ジョブ投入・キャンセルは REST API（POST / DELETE）で実行
