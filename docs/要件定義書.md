# 要件定義書

## 1. プロジェクト概要

本システムは、FastAPI をベースとした Web アプリケーションであり、日常的なタスク管理と非同期バックグラウンドジョブの管理・監視を目的とする。

---

## 2. 目的・背景

- シンプルな TODO 管理機能を提供し、タスクの作成・更新・削除・フィルタリングを可能にする
- 時間のかかる処理（外部 API 呼び出し、データ変換など）を非同期ジョブとしてキューイングし、進捗をリアルタイムで監視できる仕組みを提供する
- FastAPI + SQLModel + Taskiq + Alpine.js の組み合わせによる、シンプルかつ本番移行可能なアーキテクチャを示す

---

## 3. ユーザー

| ユーザー種別 | 説明 |
|---|---|
| 一般ユーザー | ブラウザから TODO の管理およびジョブの投入・監視を行う |
| 開発者 | API を直接利用してシステムを拡張する |

---

## 4. 機能要件

### 4.1 TODOリスト機能

| ID | 要件 |
|---|---|
| F-TODO-01 | TODO を新規作成できる（タイトル必須、最大200文字） |
| F-TODO-02 | TODO の一覧を作成日時の昇順で取得できる |
| F-TODO-03 | TODO のタイトルを編集できる |
| F-TODO-04 | TODO の完了状態を切り替えられる |
| F-TODO-05 | TODO を削除できる |
| F-TODO-06 | 「すべて / 未完了 / 完了」でフィルタリングできる |
| F-TODO-07 | タイトルをインライン編集できる（Enter 確定、Escape キャンセル） |

### 4.2 ジョブ管理機能

| ID | 要件 |
|---|---|
| F-JOB-01 | ジョブ種別とパラメータを指定してジョブを非同期投入できる |
| F-JOB-02 | ジョブ一覧を作成日時の降順で取得できる |
| F-JOB-03 | ジョブのステータス（pending / running / success / failed / cancelled）を確認できる |
| F-JOB-04 | ジョブの進捗（0〜100%）をリアルタイムで確認できる |
| F-JOB-05 | pending または running 状態のジョブをキャンセルできる |
| F-JOB-06 | ジョブの完了結果またはエラー内容を確認できる |
| F-JOB-07 | 管理画面がジョブ状態を自動更新する（SSEによるリアルタイム配信） |

---

## 5. 非機能要件

| 区分 | 要件 |
|---|---|
| **可用性** | ワーカープロセスが停止した場合でも API サーバーは継続稼働する |
| **拡張性** | 新しいジョブ種別を `app/tasks/` にタスク関数を追加し、`TASK_REGISTRY` に登録するだけで追加できる |
| **永続性** | ジョブ情報は DB に永続化されるため、ワーカー再起動後も履歴が保持される |
| **テスタビリティ** | すべての API エンドポイントはインメモリ SQLite を使ったテストで検証可能 |
| **ローカル開発** | `docker compose up` 1コマンドで全サービスを起動できる |

---

## 6. 画面要件

| 画面 | URL | 概要 |
|---|---|---|
| TODOリスト | `/` | TODO の CRUD とフィルター |
| ジョブ管理 | `/jobs.html` | ジョブ投入・一覧・リアルタイム監視 |
| API ドキュメント | `/docs` | Swagger UI による API 仕様確認 |

---

## 7. 制約・前提条件

- Python 3.11 以上
- uv によるパッケージ管理
- Redis が起動していること（ジョブ機能に必須）
- TODO 機能は Redis なしでも動作する（SQLite のみで完結）
- サーバー再起動時に `running` 状態のジョブはキャンセル扱いにはならない（タスクはRedisキューに残る）
