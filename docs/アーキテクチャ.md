## 3. ジョブのライフサイクル

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant UI as jobs.html<br/>(Alpine.js)
    participant API as FastAPI
    participant DB as SQLite
    participant Redis
    participant Worker as Taskiqワーカー

    User->>UI: ジョブ投入ボタン押下
    UI->>API: POST /api/jobs {name, params}
    API->>DB: Job作成 (status=pending)
    API->>Redis: タスクをキューに追加 (broker.kiq)
    API-->>UI: 201 Created {id, status=pending}

    loop SSE (1秒ごと)
        UI->>API: GET /api/jobs/stream
        API->>DB: ジョブ一覧取得
        API-->>UI: data: [{status, progress, ...}]
    end

    Worker->>Redis: タスク取得
    Worker->>DB: status=running, started_at=now
    loop 1秒ごと（N回）
        Worker->>DB: progress更新
        Worker->>DB: キャンセル確認
    end
    Worker->>DB: status=success, result=..., finished_at=now

    UI-->>User: 進捗バー・ステータスがリアルタイム更新
```

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant UI as jobs.html
    participant API as FastAPI
    participant DB as SQLite
    participant Worker as Taskiqワーカー

    User->>UI: キャンセルボタン押下
    UI->>API: DELETE /api/jobs/{id}
    API->>DB: status=cancelled, finished_at=now
    API-->>UI: 204 No Content

    Note over Worker: 次のループで DB をチェック
    Worker->>DB: Job取得 → status=cancelled を検知
    Worker-->>Worker: 処理を中断してreturn
```

## 7. スケールアップ時の移行方針

### DB を PostgreSQL へ移行

`.env` の `DATABASE_URL` を変更するだけで動作する（SQLModel が吸収）。

```
DATABASE_URL=postgresql+psycopg2://user:password@host:5432/dbname
```

### ワーカーをスケールアウト

`docker compose` でワーカーのレプリカ数を増やす：

```bash
docker compose up --scale worker=4
```

### ジョブ種別の追加

`app/tasks/` にタスク関数を追加し、`TASK_REGISTRY` に登録するだけでAPIから利用可能になる（詳細は[仕様書](仕様書.md#4-新しいジョブ種別の追加方法)参照）。
